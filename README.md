# Custom Block Allocator (O(1) Pool Manager)

Эффективная реализация аллокатора памяти на языке C, оптимизированная для встраиваемых систем (8/16/32-битных архитектур).

## Особенности
- **Скорость O(1)**: Выделение и освобождение памяти за константное время.
- **Zero Metadata Overhead**: Отсутствие накладных расходов на хранение размера блока рядом с данными.
- **Безопасность выравнивания**: Использование `union` и `long` гарантирует корректную работу на любых архитектурах.
- **Профилирование**: Встроенный тест производительности для сравнения с системным `malloc`.

## Структура проекта
- `custom_malloc.h` — интерфейс аллокатора и документация Doxygen.
- `custom_malloc.c` — реализация логики Free List и статических пулов.
- `main.c` — функциональные тесты и бенчмарк.

## Как запустить
Для получения корректных результатов профилирования рекомендуется использовать флаг оптимизации `-O2`:
```bash
gcc -O2 main.c custom_malloc.c -o allocator_test && ./allocator_test
```

## Технические детали (Архитектура)
1. **Free List**: Свободные блоки организованы в стек (LIFO). Указатель на следующий блок хранится прямо в свободной памяти блока, что экономит место.
2. **Определение размера**: Тип блока определяется по его адресу (вхождению в диапазон статического массива), что позволяет не хранить "заголовки" блоков.
3. **Портируемость**: Использование `uintptr_t` и выравнивание через `union` позволяют коду работать на системах с разной разрядностью без изменений.
